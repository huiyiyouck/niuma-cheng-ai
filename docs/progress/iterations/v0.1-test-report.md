## 自测范围（四维）

### 正常路径
- `POST /v1/runs/news-l1` 合法证据 → 200 + `succeeded` + 完整 `L1Output`（四维评分 / 五类标签 / 摘要 / 翻译）。
- 内部 registry dispatch `news-l1` 正确。
- 证据不足且有 URL → 触发 link read 并计数；Tavily 已配置且证据不足 → 触发 web search 并计数。
- 多 provider：首选 provider 成功即返回。

### 边界值
- 越界 `score`（9 / -2）→ clamp 到 0-5。
- provider 配置：多 provider JSON / 单 provider 兼容 / 空配置（→ `UnconfiguredClient`）。
- LLM 空响应、JSON 前后夹文本、markdown fence 包裹 → 修复解析。
- 上下文充分性阈值：`raw_text` 长文（>300 字）跳过工具；短文触发。
- 预取上下文存在 → 不触发对应主动工具。

### 异常路径
- 全部 provider 失败 → `AllProvidersFailedError` → `status=failed` + `output=null` + `error`。
- link / web 抓取失败 → recoverable 降级，`succeeded` + `tags.processing` 含 `degraded:*`。
- 不可恢复 JSON → `JSONParseError` → fallback 下一 provider。
- 非法 `LLM_PROVIDERS_JSON` / 缺必填字段 → `ProviderConfigError`。
- 未知 task_type → `UnknownTaskError`（内部错误，不暴露为新 HTTP 路由，通用路径 404）。

### 状态转换
- `succeeded` / `failed` 语义对齐 AC-6：部分可用（工具失败 / backup 命中 / 证据不足）均 `succeeded`；完全不可用才 `failed`。
- `needs_context`：工具后证据仍不足 → true（主动 KB 未落地缺口标记）。
- provider quirk：`response_format` / `temperature` 不支持 → 同 provider 调整参数重试一次。
